# PROJECT PAGE SPEC

## Format
No-scroll, single-viewport page. Every project gets this same template regardless of depth. It should feel complete on its own — not a preview, not a teaser. Think museum placard: controlled, dense, sufficient.

## Elements

**Image carousel** — 3-5 images, manually advanced (click/tap/swipe, no auto-advance). Slow, tidal transitions between images (400-600ms, ease-out). Each image can have a brief label that changes with it (1-3 words: "final UI", "early sketch", etc). This is the visual anchor of the page.

**Title** — project name, set large, given room.

**Description** — 2-4 sentences. What it is, what you did, why it matters. Written with economy. This IS the description, not a teaser for more.

**Metadata** — date, role, context. Small, quiet, sage or reduced opacity.

**Links (optional)** — 0-3 links. "Read full case study →", "View live →", "Read article →", etc. Some projects have these, some don't. The page doesn't feel empty without them.

## Principles
- Dense but controlled — everything is considered, nothing is filler.
- The composition does the work, not content volume.
- Follows BRAND_REFERENCE.md for palette, type, motion, and spatial logic.
- Bengara appears sparingly — maybe on hover over links, or the active carousel indicator.
- The page should feel like a room, not a card.
